# 강의 노트북 작성 원칙 (ch04 기준, 전체 챕터 적용)

## 1. 설명 스타일
- **어려운 내용을 학부생이 이해할 수 있을 정도로 쉽고 직관적으로 단순화한다**
- 수학 기호·수식 최소화, **일상 예시**로 먼저 설명한 뒤 개념 도입
- 전문 용어: **한글 먼저, 영어는 괄호** (예: "숨은 원인 (교란변수, Confounder)")
- 각 핵심 개념에 **"한 줄 요약"** 제공
- 비유·비교 적극 활용 (서울→부산 길, 우산과 비 등)

## 2. 준실험 설계 범위
- **DID, RDD만 다룸** (IV/도구변수법은 삭제)
- DID·RDD 각각에 ASCII 다이어그램 + 핵심 가정 명시
- 시뮬레이션 코드 셀 제공 (DID: 정책 전후, RDD: 장학금 80점 기준)
- 학습목표·마무리 등 IV 참조 일괄 제거

## 3. 실습 파트: 바이브코딩 방식
- 상세 단계별 질문(질문1~5) ❌ → 아래 구조로 통일:
  1. **시나리오** (비즈니스 맥락)
  2. **"AI에게 이렇게 질문하세요"** 프롬프트 1개
  3. 데이터 로드 코드 셀
  4. 빈 코드 셀 3개 (✏️ 1단계 / 2단계 / 3단계)
  5. **참고** (정답값·도전과제)
- 학생 흐름: AI에게 일반 질문 → AI가 방법·순서 제안 → 순서대로 코드 요청+실행+해석 요청

## 4. 시각화 원칙
- 그래프 내 모든 텍스트 `color='black'` 통일 (`textfont`, `font` 모두)
- Plotly 사용, 흑백 기조 (marker: white+black border, line: black/gray)

## 5. 구조·서식
- 파트 간 `---` 구분선
- 비교 항목은 **마크다운 표(table)** 활용
- 인과 구조는 **ASCII DAG** (코드블록 ``` 내)
- 시뮬레이션 코드 셀 하단에 `print()`로 핵심 해석 출력
- 노트북 셀 타입: 이론 = markdown, 코드 = code, 과제 제출란 = markdown
